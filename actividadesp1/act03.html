1. Introducción Contextual

Iptables es una herramienta de firewall a nivel de kernel en sistemas Linux que permite interceptar y manipular paquetes de red. En esta práctica se profundiza en el flujo de los paquetes a través de las tablas (Filter, NAT, Mangle, Raw) y las cadenas (INPUT, OUTPUT, FORWARD), elementos cruciales para la defensa perimetral de cualquier servidor.

2. Desarrollo Técnico: Tablas y Comandos

Se realizó la identificación de las tablas principales y su propósito en la seguridad de red:

  • FILTER: Tabla por defecto para el filtrado de paquetes.
  • NAT: Traducción de direcciones (Port Forwarding).
  • MANGLE: Modificación de calidad de servicio (QoS).

Traducción de Reglas de Seguridad:

Un ejemplo clave analizado en la práctica permite el tráfico web seguro (HTTP/HTTPS) y SSH en una sola regla técnica:

iptables -A INPUT -i eth0 -p tcp -m multiport --dports 22,80,443 \
-m state --state NEW,ESTABLISHED -j ACCEPT

Interpretación: Esta regla permite conexiones entrantes por la interfaz eth0 hacia los puertos de administración (22) y servicios web (80, 443), siempre que sean conexiones nuevas o ya establecidas.

4. Opinión y Reflexión Técnica

Dominar la sintaxis de iptables es vital porque, a diferencia de un firewall comercial con interfaz gráfica, trabajar directamente con el kernel de Linux nos da un control absoluto sobre el tráfico. En esta práctica aprendí que un error en el orden de las reglas (específicamente con la política DROP) puede dejar incomunicado un servidor completo. La seguridad empieza por entender cómo se mueve cada bit en la red.
