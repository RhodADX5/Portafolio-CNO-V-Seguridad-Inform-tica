1. Introducción Contextual

Para garantizar la integridad en las interconexiones digitales, existen marcos de referencia fundamentales. El estándar ITU-T X.800 define la arquitectura de seguridad para el modelo OSI, mientras que el RFC 4949 funciona como un glosario exhaustivo que estandariza la terminología de seguridad en Internet. En esta actividad, se analizan 10 escenarios de riesgo para identificar qué servicios y mecanismos fueron vulnerados.

2. Desarrollo Técnico: Análisis de Escenarios

Se aplicó el modelo X.800 para diagnosticar incidentes comunes. Aquí algunos ejemplos clave del análisis:

Escenario: Intercepción de Mensajes

Descripción: Un atacante captura correos electrónicos entre dos gerentes.

  • Servicio afectado: Confidencialidad de datos.
  • Mecanismo de seguridad: Cifrado (Encryption).

Escenario: Modificación de Transacciones

Descripción: Se altera el monto de una transferencia bancaria en tránsito.

  • Servicio afectado: Integridad de datos.
  • Mecanismo de seguridad: Firma digital / Integridad de datos.

4. Opinión y Reflexión Técnica

Desde mi perspectiva en este 9no semestre, el estudio de X.800 y RFC 4949 es la base de cualquier estrategia de defensa. Entender la diferencia entre un ataque pasivo (intercepción) y uno activo (modificación) permite al ingeniero seleccionar los controles adecuados. Sin una terminología estándar (RFC 4949), la comunicación en un equipo de respuesta a incidentes sería caótica y poco efectiva.
