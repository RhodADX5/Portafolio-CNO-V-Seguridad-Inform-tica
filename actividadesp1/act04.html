1. Introducción Contextual

La defensa en red se basa en el principio de privilegio mínimo. En esta actividad, se diseñó un conjunto de reglas de filtrado para un Firewall que protege una red local con servidores públicos (DMZ). El objetivo es permitir únicamente el tráfico legítimo (DNS, Web, Correo) y denegar todo lo demás por defecto, minimizando la superficie de ataque.

2. Desarrollo Técnico: Configuración del Firewall

Basado en la topología de red (Internet -> Firewall -> Servidores), se establecieron las siguientes políticas de iptables:

A. Políticas Base y Estado

# Establecer política restrictiva
iptables -P INPUT DROP
iptables -P FORWARD DROP

# Permitir tráfico de conexiones ya establecidas
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

B. Reglas para Servidores Críticos

Para el Servidor Web (192.1.2.11) y el Servidor de Correo (192.1.2.10), se habilitaron los puertos estándar garantizando que el origen y destino sean precisos:

  • Tráfico DNS (TCP/UDP): Permitir consultas salientes de la red local.
  • Correo Entrante (SMTP): Apertura del puerto 25 para el servidor 192.1.2.10.
  • Acceso Web: Tráfico HTTP/HTTPS para el servidor 192.1.2.11.

4. Opinión y Reflexión Técnica

Esta práctica me permitió comprender la importancia de la inspección de estados (Stateful Inspection). No basta con abrir un puerto; es crucial entender que una conexión establecida debe ser permitida de regreso automáticamente. Al configurar el Firewall para los servidores de correo y web de la UPSLP (en el escenario propuesto), aprendí que la precisión en las máscaras de subred y las IPs de origen evita el "spoofing" y otros ataques de suplantación comunes en la capa de red.
